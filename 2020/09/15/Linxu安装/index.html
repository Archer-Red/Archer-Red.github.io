Linxu安装踩坑

发表于 2020-09-15 更新于 2020-09-16 分类于 安装笔记
linux安装过程中的一系列问题

linux安装教程参考

参考教程

  • 安装步骤
    1.分配磁盘空间，>=100G,制作启动盘，工具rufus
    2.安装ubantu，安装类型一定选择其它选项
    3.分区操作
      - 挂载在/， 根目录，文件为ext4，主分区, 也就是 ubuntu 的根目录就一个反斜杠表示，相当于windows的C盘， 我们将其分为 10G，文件格式为 ext4，条件允许可以大一点，可以30G，毕竟ubuntu装软件默认是装在这里的，大一点可能会省去后面隐藏的麻烦。
  - 挂载在/boot,逻辑分区，这个就是实现你双系统的原因了，这个就是用于启动 ubuntu 的目录，里面会有系统的引导，这个文件其实只有几十兆，但是我们建议将其划分为 400M 文件格式为 ext4，这个分区必不可少，否则后果你懂得！
  - 交换空间 swap，逻辑分区，这个是 Linux 也就是 ubuntu 的交换区目录，这个一般的大小为内存的 2 倍左右， 主要是用来在电脑内存不足的情况下，系统会调用这片区域，来运行程序，我们可以将其分为 10G， 这个把ext4换成交换空间，英文swap
  - 挂载点，/home:这是 ubuntu 的“其他盘”， 这个也可以说是我们的个人目录，相当于windows的其他盘，所以为了让我们自己的目录大一点，剩下的全分给它，文件格式为 ext4
  - bios预留分区，可留也可不留，不分会报错，但是直接继续应该会自动分。
    4.注意安装启动器是设备一定是/boot所在的分区，继续安装完成

    踩坑

1.注意安装前修改bios设置，将security boot设置为disable，才能写入引导区
2.安装注意自己电脑的bios的引导模式，可以在win+r输入msinfo32查看，安装linux引导必须可windows的引导模式相同，在bios setup里面修改，ps一般一样，我就是手欠修改了，然后修改了一下午，联想电脑设置u盘启动盘可能要修改UEFI模式为legacy fist模式，如果这样这样安装，电脑哪怕安装好linux，但是引导界面一定不对，而且网上的办法很难修复
3.修复引导界面方法
- ctrl+alt+t打开终端，输入$ sudo update-grub,更新后看见windows则成功
- 不行的话安装，bootrepair
```sudo add-apt-repository ppa:yannubuntu/boot-repair
sudo apt-get update
sudo apt-get install -y boot-repair && boot-repair
4.sudo提示xx行有语法错误，可能是修改了/etc/sudoers。tmp文件，办法在于进入//etc/目录，输入pkexec visudo 然后修改对于的行
5.安装nodejs，[参考教程][https://www.cnblogs.com/feiquan/p/11223487.html]
6.npm安装软件包，需要sudo]
7.hexo的deploy已经安装，发生仓库fatal，输入yes，输入y报错
8.sudo rm /var/lib/dpkg/lock-frontend 无法安装出现前端锁,删除其他的锁
9.安装Aacaonda‘安装包不能改名字
10.更换介质：请把标有

“Ubuntu-Kylin 14.04 LTS Trusty Tahr - Release amd64 (20140417.1)”
的盘片插入驱动器“/media/cdrom/”再按回车键

解决办法：

更改软件源，在设置中有个软件与更新，在里面将从光驱安装取消掉，然后终端执行sudo apt-get update和sudo apt-get upgrade
